What is Git hub?


*Version control a system that allows you to save the specific version of your work without affecting the rest of the group's work on a project *Repository a folder that is a storage space for the files needed for a project *Commit A snapshot of your repo at a certain time *Branch a separated line of development in a repo
Git Commands
    Git init Creates an empty repository Git clone Clones a repository into a new directory *Git add Updates the index using the content in the working tree *Git commit Saves the state of your project at that time point *Git push Uploads your files available on your CPU to the repo in the cloud *Git Pull Updates the files uploaded into the cloud to your CPU *Git branch Breaks away from the mainline for separate work
